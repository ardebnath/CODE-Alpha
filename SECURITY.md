# Security Policy

This repository is maintained under the `Bluear_cod` brand.

## Supported Security Scope

Security reports are welcome for the current project files, especially:
- Alpha interpreter runtime
- local browser servers
- desktop software
- account and project save/load behavior
- local file handling
- package manager behavior

## How To Report A Security Issue

Please do not post new security vulnerabilities publicly in an issue first.

When reporting a vulnerability, include:
- affected file or folder
- steps to reproduce
- what could go wrong
- screenshots or logs if available
- your suggested fix if you have one

## Initial Maintainer Goal

The goal is to review good-faith security reports as quickly as possible and patch confirmed problems in a later update.
