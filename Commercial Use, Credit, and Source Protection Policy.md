# Commercial Use, Credit, and Source Protection Policy

Project name:
- `Alpha`

Project owner / brand:
- `Bluear_cod`

## Important Legal Note First

This document is a policy draft for how the project owner wants Alpha to be used going forward.

Important:
- this document does **not automatically revoke** rights already granted under any earlier public or open-source release
- if earlier copies of Alpha were already published under a license like `MIT`, people who already received those copies may still keep those earlier license rights for those copies
- stronger future enforcement still depends on keeping these terms in the main repository license files and getting legal review if needed

This means:
- this policy is useful as a business and repository rule document
- but it should not be treated as guaranteed retroactive legal cancellation of old public license grants

## Policy Goal

The goal of this policy is:
- to let people learn from Alpha
- to require visible credit to `Bluear_cod`
- to require a paid license for business or large-scale use
- to protect the core Alpha code from copying, republishing, resale, or reuse in another main product without written permission

## Allowed Personal And Learning Use

People may use Alpha for:
- personal learning
- education
- private experiments
- local testing
- non-commercial demonstrations

These users should still keep the project notices and credits.

## Required Credit

Any allowed public use of Alpha should give visible credit to:

`Alpha by Bluear_cod`

Recommended credit line:

`Powered by Alpha by Bluear_cod`

Credit should appear in one or more of these places:
- README
- documentation
- about page
- credits page
- footer
- release notes

## Commercial And Business Use

Commercial, business, client, enterprise, paid, or large-project use requires:
- prior written permission from `Bluear_cod`
- a paid commercial license or written commercial agreement

Examples that should require a paid license:
- using Alpha inside a paid software product
- using Alpha in SaaS, client work, or internal company platforms
- using Alpha in a business training platform with paid users
- shipping Alpha as part of a commercial desktop app, mobile app, or hosted service
- using Alpha in a major school, company, institution, or production deployment

## Large Project Use

For this policy, a "big project" or "large project" includes things like:
- public production systems
- company internal platforms
- client-delivery projects
- platforms with many users
- paid education platforms
- enterprise or organizational use

If there is doubt, the user should treat the project as commercial use and request written permission first.

## No Copying Of Main Code

Without written permission from `Bluear_cod`, people should **not**:
- copy the main Alpha source code into another core product
- republish the core Alpha source as their own project
- remove copyright or branding notices
- sell, sublicense, or redistribute the core Alpha code
- rebrand the core Alpha code as a different main language product
- copy substantial parts of the core Alpha code into a competing platform

## Core Code Covered By This Policy

The "main code" or "core code" especially includes:

## Allowed Reference Use

Unless a different written license is granted, people should treat Alpha's core code as:

- not freely copyable into other main products or commercial systems
